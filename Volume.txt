
★★볼륨의 목적
특정노드에 포드가 갈시 다른 노드에서 사용 불가능, 이를 해결하여 다른 노드에서도 해당 포드를 사용하기 위해 볼륨 정의


+++ hostpath = 호스트와 볼륨 공유
+++ emptydir 포드의 컨테이너 간에 볼륨을 공유


[ 네트워크 볼륨 ]
NFS ( 1:N 여러개의 인스턴스 마운트 가능)
- 네트워크 상에서 파일 시스템을 공유
- 여러개의 클라이언트 동시 마운트 ( 로컬 스토리지처럼 사용할수 있다는 장점 )
* 사용을 위해선 1) NFS 서버(실제 데이터 저장 스토리지 서버,영속적 저장) 
                    2) NFS 클라이언트(NFS 서버에 마운트해 스토리지에 파일을 읽고 쓰는 역할) - 워커 노드의 기능을 사용

퍼시스턴트 볼륨 ( PV ) 과 퍼시스턴트 볼륨 클레임 ( PVC )

( 사용자는 디플로이먼트의 YAML 파일에 볼륨의 상세한 스펙을 정의하지 않아도 된다 )
* 외부 볼륨을 필요로 한다는 사실만을 포함, 볼륨이 무엇인지는 정의하지 않음
